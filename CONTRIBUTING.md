# Contributing

Thank you for your interest in `py-overload` (`strongtyping_pyoverload`).

## TL;DR

**This project does not accept pull requests.** This includes pull requests generated
wholly or in part by AI coding agents (GitHub Copilot, Codex, Cursor, Devin, Claude Code,
Junie, Aider, and any similar tools). Such PRs will be closed without review.

Please use the **issue tracker** instead — see below.

Under the "NoDerivatives" term of that license, and as restated in the *Reporting Changes*

> Direct modifications to the source code by anyone other than the maintainers are
> prohibited under the "NoDerivatives" term of this license.

For this reason, all source code changes are made exclusively by the maintainers.

## How to contribute

You can still help the project a lot — just not by submitting code directly.

### 1. Report a bug

Open a **Bug Report** in the
[issue tracker](https://github.com/FelixTheC/py-overload/issues) with:

- A clear, descriptive title.
- The version of `strongtyping_pyoverload` and Python you are using.
- A **minimal, reproducible** code snippet that triggers the bug.
- The expected behavior and the actual behavior (including full traceback if any).

### 2. Request a feature

Open a **Feature Request** in the
[issue tracker](https://github.com/FelixTheC/py-overload/issues) describing:

- The problem or use case you are trying to solve.
- Why the existing API is not sufficient.
- A rough sketch of the API you would like to see (optional but appreciated).

### 3. Improve documentation feedback

If you spot a typo, an unclear paragraph, or a missing example in `README.md`,
`HowTo.md`, `blog.md`, or anything under `docs/`, please open an issue describing
the location and the suggested wording. The maintainers will apply the change.

## Pull requests

Pull requests opened against this repository — including those opened by automated
or AI-driven tools (Dependabot, Renovate, Copilot agents, Codex, Cursor background
agents, Devin, Claude Code, Junie, Aider, etc.) — will be **closed without review**.

This is not personal — it is a direct consequence of the license under which this
project is published. If you have already prepared a change locally, please copy the
*description* of the change into an issue. The maintainers can then evaluate and,
if appropriate, implement it.

## AI-generated contributions

To be explicit: contributions (in any form, including issue text, code snippets in
issues, or PRs) that are generated by AI tools must be:

1. Clearly disclosed as AI-generated in the issue body, and
2. Reviewed by a human (you) for correctness before submission.

PRs that appear to be drive-by AI-generated patches will be closed immediately.

## Code of conduct

Be respectful. Off-topic, abusive, or spammy issues will be closed and the author
may be blocked from the repository.

## Questions

For anything that does not fit a bug report or a feature request, please open a
**Discussion** (if enabled) or a regular issue tagged `question`.
